# 業務要件定義書

## 1.はじめに

### 1-1.システムについて

「学習レポート共有サイト」は各個人が行っている学習のまとめ等を、PDF形式ファイルに変換して投稿（シェア）を行うことができるサイト。<br>
学習成果をアウトプットしたり、他の人の学習成果からヒントを得て自分自身の学習に活用していくことができる。

### 1-2.システム化の目的

本システムを通して、福祉事業所内の利用者同士のコミュニケーションを取る手段を増やす。<br>
学習レポートのシェアを通して、教える、教わる、共感する等会話のフックを確立していく。<br>
学習レポートの蓄積から学習ナレッジ貯め、事業所全体の技術向上を図る。

## 2.概要

### 2-1.現行の学習ナレッジの共有方法

- 朝礼時の3分間スピーチやGood&Newでの発信
- 支援員からの共有

### 2-2.次期の学習ナレッジの共有方法

- 従来の方法
- **「学習レポート共有サイト」を通した、利用者の自発的な受発信**

## 3. 業務要件定義

### 3-1. システム概要

本システムは、学習レポートの投稿と閲覧が可能な会員制サイトである。<br>
サイトの投稿と閲覧には、管理者アカウントより発行されたアカウントでログインする必要がある。<br>
本サイトの閲覧には、システムに登録しているGoogleアカウントにログインしているブラウザを使用する必要がある。

### 3-2. ユーザー権限

本システムで使用するアカウントは、以下の権限別ユーザーとする。<br>
|#|ロール|用途|
|:----|:----|:----|
|1|支援員|支援員が利用者を登録、管理するアカウント|
|2|利用者|利用者が学習レポートの投稿、閲覧を行うアカウント|
|3|ゲスト|一時的に閲覧のみ可能なアカウント|

### 3-3. 業務要件

本システムでは、以下の機能で業務要件達成する。<br>
|#|分類|機能|要件|
|:----|:----|:----|:----|
|1|認証|ログインする|・ログインIDとパスワードでログインを用いること|
| | | |・事前登録されている情報と一致したときのみログイン|
| | | |・事前登録の内容と異なる場合はログイン画面にリダイレクト|
|2| |ログアウトする|・ボタン押下でセッションアウトすること|
| | | |・ログアウト画面を表示すること|
|3|アカウント|アカウント一覧を取得する|・登録済みのアカウントをロールごと、最終更新日順に取得する|
| | | |・パスワードは取得情報に含ませないこと|
|4| |アカウント詳細を取得する|・1件のアカウントID（登録時に一意の自動採番ID）から登録情報を取得する|
|5| |アカウントを登録する|・新規にユーザーアカウントを登録する|
| | | |・一意のアカウントIDを自動採番する|
| | | |・ログインID、アカウント名（表示名）、メールアドレス（Google限定）、ロール、パスワードの登録を必須とする|
| | | |・ログインIDの重複は許容しない|
| | | |・同一メールアドレスの登録は許容しない|
| | | |・メールアドレスに対して、オーナーアカウントのGoogleドライブ内の専用フォルダの閲覧権限を付与する|
|6| |アカウントを更新する|・アカウント名（表示名）、メールアドレス（Google限定）、ロール、パスワードの更新を行う|
| | | |・ログインIDの更新は許容しない|
| | | |・パスワードは変更があるときのみ更新項目に含め、更新対象外の時は更新情報に含めない|
| | | |・同一メールアドレスの登録は許容しない|
| | | |・メールアドレス変更時は、オーナーアカウントのGoogleドライブ内の専用フォルダの閲覧権限を旧メールアドレスから剥奪し新メールアドレスに付与する|
| | | |・他に同一メールアドレスを使用しているアカウントがある場合は、オーナーアカウントのGoogleドライブ内の専用フォルダの閲覧権限の付与を旧メールアドレスに対して継続する。|
|7| |アカウントを削除する|・削除前に確認メッセージを表示すること|
| | | |・削除と同時にデータベース上からも完全削除を行う|
| | | |・削除後の復元は不可能とする|
| | | |・削除後に同一情報で再登録は可能とする|
| | | |・当該アカウントのメールアドレスから、オーナーアカウントのGoogleドライブ内の専用フォルダの閲覧権限を剥奪する|
|8|学習レポート|学習レポート一覧を取得する|・登録済みの学習レポート一覧を取得する|
|9| |学習レポートを登録する|・新規学習レポート登録を行う|
| | | |・一意の学習レポートIDを自動採番する|
| | | |・タイトル、投稿者名（イニシャル等）、PDFファイル添付を必須項目とする|
| | | |・概要は任意項目とする|
| | | |・公開日は登録日を自動設定する|
| | | |・公開終了日は登録月の3か月後の月末を自動設定する|
| | | |・PDFファイルはオーナーアカウントの権限で、オーナーアカウントのGoogleドライブに登録し、ファイルIDを取得してデータベースに登録する|
| | | |・PDFファイルはアクセス制限付きファイルとしてファイルの共有を許容する（公開範囲はフォルダのアクセス権限のある者）|
|10| |学習レポートを更新する|・タイトル、投稿者名（イニシャル等）、PDFファイル添付、概要の更新を行う|
| | | |・タイトル、投稿者名（イニシャル等）、PDFファイル添付を必須項目とする|
| | | |・概要は任意項目とする|
| | | |・公開日の変更は許容しない|
| | | |・PDFファイルは更新があった時のみ、旧ファイルを削除し、新ファイルを再登録して、ファイルIDを取得してデータベースを書き換える|
|11| |学習レポートを削除する|・削除前に確認メッセージを表示すること|
| | | |・削除と同時にデータベース、Googleドライブ上のファイルも完全削除を行う|
| | | |・削除後の復元は不可能とする|
| | | |・削除後に同一情報で再登録は可能とする|
|12|タグ|タグ一覧を取得する|・タグ名一覧を取得する|
|13| |タグの登録をする|・タグ名を登録する|
| | | |・タグ名の重複は許容しない|
|14| |タグの更新をする|・タグ名を更新する|
| | | |・タグ名の重複は許容しない|
|15| |タグの削除をする|・タグを削除する|
|16|ジョブ|月次ジョブ|・公開終了日を過ぎた学習レポートをデータベースとGoogleドライブ上のファイルから削除する|
